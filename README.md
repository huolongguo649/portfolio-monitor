# 个人持仓监控桌面工具

赛博数据风桌面持仓监控应用，基于 Electron + Vue3 + ECharts。

## 功能说明

### 1. 资产总览 (左侧面板)
- 五大核心指标: 总资产 / 持仓市值 / 可用资金 / 今日盈亏额 / 盈亏%
- 可用资金可直接在面板中修改
- 持仓健康度环形图 (0-100分)，颜色随分数变化
- 健康度详情逐条展示扣分原因

### 2. 持仓明细 (中间面板)
- 表格展示所有持仓的完整信息
- **现价可直接在表格中修改** (无需弹窗)
- 数量、行业也支持直接编辑
- 点击表头排序 (代码/名称/数量/成本价/市值/盈亏/占比)
- 每行占比带进度条，超 20% 红色预警
- 盈亏状态自动标注 (红涨绿跌)

### 3. 行业分布 (右侧面板)
- 饼图展示行业/板块分布
- 前三大持仓展示
- 集中度超 50% 时醒目提示

### 4. 预警提醒
- 涨跌幅触发 (默认 ±5%)
- 单只占比超限 (默认 20%)
- 非阻塞方式显示在面板中，不弹窗

### 5. 数据管理
- 导入 CSV/Excel (自动识别常见表头)
- 手动添加持仓
- AES-256 加密保存为 .dat 文件
- 密码解密加载
- 导出 CSV 快照

## 数据导入说明

### 支持格式
- **CSV** (.csv) — UTF-8 编码
- **Excel** (.xlsx / .xls) — 取第一个 Sheet

### 表头命名规则

系统自动识别以下表头（大小写不敏感）：

| 字段 | 支持的表头名 |
|------|-------------|
| 代码 | 代码、证券代码、股票代码、基金代码 |
| 名称 | 名称、证券名称、股票名称、基金名称 |
| 数量 | 持仓数量、数量、股数、份额 |
| 成本价 | 成本价、成本、持仓成本、买入均价 |
| 现价 | 现价、最新价、当前价、收盘价 |
| 行业 | 行业、板块、所属行业、行业分类 |

### CSV 示例

```csv
代码,名称,持仓数量,成本价,现价,行业
000001,平安银行,1000,10.50,10.80,银行
600519,贵州茅台,100,1650.00,1680.50,白酒
300750,宁德时代,200,198.50,195.20,新能源
```

参考 `导入模板.csv` 文件。

### 未匹配列处理
系统会提示哪些列未被识别，不影响已识别列的导入。

## 关键设计决策

### 为什么不接实时行情 API？
- 个人投资者不需要毫秒级实时数据
- 避免 API 费用和合规问题
- 现价可手动修改或通过重新导入 CSV 批量更新
- 简化技术架构，降低维护成本

### 为什么用 AES-256 加密？
- 持仓数据属于敏感财务信息
- AES-256 是工业级加密标准
- crypto-js 纯 JS 实现，无需原生依赖
- 加密后的 .dat 文件即使泄露也无法解密

### 为什么用 Pinia 而非 Vuex？
- Pinia 是 Vue 官方推荐的状态管理方案（Vuex 5 的后继者）
- Composition API 风格，与 Vue3 `<script setup>` 一致
- 完整的 TypeScript 支持（可平滑迁移）
- 更轻量，无 mutations 概念

### 健康度算法设计
- 评分规则写在 `src/utils/health.js`，可自行修改
- 100 分满分，4 个扣分维度
- 单向扣分（不隐藏问题），最低 0 分
- 预留了扩展接口，可添加自定义评分规则

### 为什么用自定义标题栏？
- 深色主题下系统标题栏不协调
- 无边框窗口最大化赛博数据风视觉效果
- Electron `frame: false` + 自定义拖拽区域

## 扩展指南

### 自定义健康度算法
编辑 `src/utils/health.js` 中的 `calculateHealthScore` 函数，添加/修改扣分规则。

### 自定义预警规则
编辑 `src/utils/health.js` 中的 `checkAlerts` 函数。

### 添加新的数据导入格式
编辑 `src/utils/parser.js`，在 `HEADER_MAP` 中添加新表头映射，在 `parseFile` 中添加新格式支持。
